# Flow: Generate Test Cases

**Actor:** Business Analyst / QA

<execution_protocol>

1. **Source:** Read the User Stories and Acceptance Criteria in `.windsurf/memory/pm/`.
2. **Draft:** Write Gherkin scenarios (`Given/When/Then`) for each acceptance criterion.
3. **Cover:** Ensure positive paths, negative paths, and edge cases are covered.
4. **Output:** Save as `.windsurf/memory/analyst/[feature]_tests.feature` (or `.md`).
   </execution_protocol>
